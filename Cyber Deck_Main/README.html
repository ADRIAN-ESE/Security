⬡ Documentation

CyberSuite

Security Toolkit — Three Editions

A browser-based security toolkit providing encryption, phishing analysis, and QR code generation — no server, no backend, all client-side.

🔐 Client-Side Only 📡 No Backend ✅ Zero Dependencies Installed 🌐 Vanilla HTML/JS

📂 Project Files

⬡ Modules

🔐
Cipher Suite
Encrypt and decrypt messages using four selectable algorithms. Key input, mode toggle, copy and swap utilities included.
🎣
PhishGuard
Paste a suspicious email or message to receive a structured analysis: risk score, identified red flags, explanation, and a security tip.
📱
QR Studio
Generate QR codes for URLs, plain text, Wi-Fi credentials (SSID + password + auth type), and vCard contacts. Live preview with download.

🔑 Cipher Algorithms

All encryption runs in the browser via CryptoJS 4.1.1. No data is transmitted.

AES-256
Modern Advanced Encryption Standard, 256-bit key. Requires a passphrase. Default algorithm — recommended for sensitive data.
XOR
Classical Bitwise XOR cipher. Output is Base64-encoded for readability. Simple and fast; not suitable for high-security use cases.
Caesar
Classical Substitution cipher with a configurable character shift (default: 3). Letters only; non-alpha characters are preserved.
Base64
Encoding Encoding/decoding only — not encryption. No key required. Useful for obfuscation or data transport formatting.

🎣 PhishGuard Detail

Paste any email, SMS, or message body into the input area. The analyzer evaluates the content against a built-in pattern library and returns:

  • Risk score — LOW / MEDIUM / HIGH / CRITICAL classification
  • Red flags — specific suspicious phrases or patterns detected
  • Explanation — plain-language breakdown of why it's suspicious
  • Security tip — contextual advice for the detected attack vector

Analysis is entirely local — no text is sent to any external service.

📱 QR Studio Detail

Powered by qrcodejs 1.0.0. Four input modes are available via sub-tabs:

  • URL — any web address
  • Text — arbitrary plain text
  • Wi-Fi — SSID, password, and security type (WPA/WEP/None) for instant network sharing
  • vCard — contact card fields (name, phone, email, org, URL)

The generated QR code renders as a live preview with a download button.

🚀 Usage

No build step. No dependencies to install. Open any file directly in a browser:

# Option 1 — open directly
open cybersuite_v5.html

# Option 2 — serve locally (avoids any browser file:// quirks)
npx serve .
# or
python3 -m http.server 8080

External CDN assets are loaded at runtime — an internet connection is required on first load for fonts and library scripts.

🛡️ Privacy & Security Notes

  • All cryptographic operations run locally in the browser via CryptoJS — no plaintext or keys leave your device.
  • PhishGuard analysis is fully offline — pasted message content is never transmitted.
  • QR code generation is client-side — no data is sent to a QR API service.
  • The only external network calls are for CDN assets (fonts, CryptoJS, qrcodejs) on page load.
